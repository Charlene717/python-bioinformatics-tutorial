CHAPTER 17 / 18

單細胞 LLMs

把每個 cell 看成一個「句子」、每個基因看成「token」— transformer 應用到 scRNA-seq 的革命。

Treat each cell as a "sentence" and each gene as a "token" — the transformer revolution in scRNA-seq.

觀念

一、scLLM 與 ChatGPT

大型語言模型（LLM, Large Language Model）的本質：給一段「token 序列」，學出有意義的 embedding。

scLLM 的核心類比：

  • 一個 cell = 一個「句子」
  • 一個 gene = 一個「字 (token)」
  • 表達量 = 該字的「強度」/ rank
  • 模型用 transformer 學出每個 cell 的 latent embedding

訓練時用大量公開 scRNA-seq 資料（千萬到億級 cells），學出泛用的「細胞語言模型」。下游任務只要 fine-tune 一下，就能做：

  • 跨資料集 batch 整合（zero-shot）
  • 細胞類型自動註解
  • 基因擾動效應預測（perturb-seq）
  • 缺失值填補 (imputation)
  • 跨物種對應

An LLM's core idea: given a sequence of tokens, learn meaningful embeddings.

scLLM analogy:

  • One cell = one "sentence"
  • One gene = one "token"
  • Expression = token strength / rank
  • A transformer learns a latent embedding per cell

Trained on tens-to-hundreds of millions of public scRNA-seq cells, the model learns a general "cell language model." With light fine-tuning it can do:

  • Cross-dataset batch integration (zero-shot)
  • Automatic cell-type annotation
  • Predict gene perturbation effects (perturb-seq)
  • Imputation
  • Cross-species mapping
主流模型

二、主流模型比較

模型作者 / 年訓練資料特色

scGPTCui et al., Nature Methods 202433M cells (multi-omics)generative transformer；支援 perturb 預測、多 omics、整合 Generative transformer; supports perturbation prediction, multi-omics, integration
scFoundationHao et al., 202350M cellsread depth-aware；非對稱 encoder/decoder；強在 small samples Read-depth aware; asymmetric encoder/decoder; strong on small samples
scBERTYang et al., 20221M cells先驅之一，Performer 架構處理長序列 Early pioneer, Performer architecture for long sequences
UCE (Universal Cell Embeddings)Rosen et al., 202336M cells, 8 種物種跨物種統一 embedding Cross-species unified embedding

⚠️
實事求是：2024–2025 的多份基準測試（Kedzierska et al., Boiarsky et al.）發現，目前 scLLMs 在 zero-shot 整合上仍常輸給傳統方法 (scVI, Harmony)。優勢主要在 fine-tune 後的 perturbation prediction 與少樣本場景。請當工具用，不要當神。 Reality check: 2024-2025 benchmarks (Kedzierska et al., Boiarsky et al.) show scLLMs often underperform classical methods (scVI, Harmony) in zero-shot integration. Their edge appears in fine-tuned perturbation prediction and low-sample regimes. Treat them as tools, not oracles.
實作

三、scGPT zero-shot 實作

# GPU 環境，安裝 / GPU env, install $ pip install scgpt # 下載預訓練模型 / Download pretrained checkpoint # https://github.com/bowang-lab/scGPT 提供多個 checkpoint (whole-human, ...) 
import scanpy as sc
import torch
from scgpt.tasks import embed_data
from pathlib import Path

adata = sc.read_h5ad("data/processed/multi_sample.h5ad")

# scGPT 預訓練 checkpoint 路徑
model_dir = Path("models/scgpt_human")

# 一行得到 cell embedding
adata = embed_data(
    adata,
    model_dir=model_dir,
    gene_col="gene_name",
    obs_to_save=["batch", "cell_type"],
    batch_size=64,
    device="cuda" if torch.cuda.is_available() else "cpu",
)
# scGPT embedding 存到 adata.obsm["X_scGPT"]

# 拿來做整合 UMAP
sc.pp.neighbors(adata, use_rep="X_scGPT")
sc.tl.umap(adata)
sc.pl.umap(adata, color=["batch","cell_type"])

# 跟 scVI / Harmony 比一比，看能不能更好移除 batch
# 用 scGPT fine-tune 做細胞註解
# Fine-tune scGPT for cell-type classification
from scgpt.tasks import annotate_data

# 準備：reference (有標籤) + query (沒標籤)
ref = sc.read_h5ad("reference.h5ad")         # 有 cell_type 欄
query = sc.read_h5ad("query.h5ad")             # 待標注

predictions = annotate_data(
    train_data=ref,
    test_data=query,
    cell_type_col="cell_type",
    model_dir=model_dir,
    n_epochs=10,
    lr=1e-4,
)
query.obs["predicted_cell_type"] = predictions
sc.pl.umap(query, color="predicted_cell_type")
# 預測 CRISPR 擾動後的表達變化
# Predict expression changes after CRISPR perturbation
from scgpt.tasks import perturb_predict

# Norman 2019 perturb-seq 資料：1 cell × 1-2 gene 敲降
adata = sc.read_h5ad("data/perturb_train.h5ad")
# adata.obs["perturbation"] = "TP53,MYC"  例如同時敲降兩個基因

model = perturb_predict.train(
    adata, model_dir=model_dir,
    n_epochs=15, batch_size=32,
)

# 預測「同時敲降 TP53+MYC」會怎樣
predicted = model.predict_perturbation(["TP53", "MYC"])
# predicted: np.ndarray (n_genes,) — 預測表達變化

四、Geneformer 工作流

# pip install git+https://huggingface.co/ctheodoris/Geneformer
from geneformer import EmbExtractor, TranscriptomeTokenizer

# 1. Tokenize 你的 AnnData → Geneformer 格式
tk = TranscriptomeTokenizer(custom_attr_name_dict={"cell_type":"cell_type"})
tk.tokenize_data("data/processed/", "output_dir/", "my_data", file_format="h5ad")

# 2. 取出 embedding
extractor = EmbExtractor(model_type="CellClassifier",
                         num_classes=0,
                         emb_mode="cell",
                         emb_layer=0,
                         max_ncells=10000)
embs = extractor.extract_embs(
    model_directory="Geneformer/model/",
    input_data_file="output_dir/my_data.dataset/",
    output_directory="results/geneformer/",
    output_prefix="embs",
)

# 3. 把 embedding 接回 AnnData 做下游
import anndata as ad
adata.obsm["X_geneformer"] = embs.values
sc.pp.neighbors(adata, use_rep="X_geneformer")
sc.tl.umap(adata)
決策

五、什麼時候用 scLLM

🌳 scLLM 何時派上用場

✅
樣本量很少 (<1000 cells) → 預訓練模型可借力大型語料
✅
要做 perturbation prediction（合成擾動效果）
✅
跨物種 / 跨組織映射 (UCE)
❌
標準 scRNA-seq 整合 → 還是 scVI / Harmony 較穩
❌
嚴格 reproducibility 場景 → 模型權重會更新、結果不穩定
⚠️
沒有 GPU → 跑會很慢，建議用 CPU 子集測試
✅
Very small datasets (<1000 cells) → leverage the pretrained corpus
✅
Perturbation prediction (synthesizing knockout effects)
✅
Cross-species / cross-tissue mapping (UCE)
❌
Standard integration → scVI / Harmony usually wins
❌
Strict reproducibility — checkpoints can update; results may shift
⚠️
No GPU → it's slow; subset for CPU testing
