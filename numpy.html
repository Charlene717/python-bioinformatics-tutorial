NumPy 數值運算

所有生資數值套件 (pandas, scanpy, scikit-learn) 都站在 NumPy 上 — 學會它就打通任督二脈。

Every numerical bioinformatics package (pandas, scanpy, scikit-learn) is built on NumPy — master it and the rest opens up.

概念

一、為什麼用 NumPy？

Python 內建 list 處理大量數值有兩大問題：慢（每個元素是物件，運算要走 Python 解譯器）和耗記憶體。

NumPy 的 ndarray（N-dimensional array）是連續記憶體＋固定型別的陣列，運算用 C 寫的，比 list 快 50–100 倍。

生資場景：基因表達矩陣 (cells × genes) 可能 10000×30000 = 3 億個數值 — 沒有 NumPy 根本跑不動。

Python's built-in list has two problems for bulk numerics: slow (each element is an object; ops go through the Python interpreter) and memory-hungry.

NumPy's ndarray is a contiguous block of memory with a fixed dtype, with operations written in C — 50–100× faster than lists.

Bioinformatics scale: an expression matrix (cells × genes) of 10000 × 30000 = 300M numbers — impossible without NumPy.

⚡
核心心法：「能向量化就絕不用 for 迴圈」。NumPy 程式好不好，看的就是 for 迴圈少不少。 Core mindset: "Vectorize, never loop." Good NumPy code is measured by the absence of for loops.

向量化運算

四、向量化 + Broadcasting

Broadcasting 是 NumPy 把不同 shape 的 array 自動「對齊」的機制。掌握 broadcasting，整個基因表達標準化只要一行。

Broadcasting is NumPy's mechanism for aligning arrays of different shapes automatically. Master it and entire normalization steps become one-liners.

a = np.array([1,2,3,4])
b = np.array([10,20,30,40])

a + b           # [11,22,33,44] — 元素相加
a * b           # [10,40,90,160] — 元素相乘 (不是矩陣乘法)
a / b           # [.1,.1,.1,.1]
a ** 2          # 平方
np.log2(a)      # log2 — 全部 element-wise
np.exp(a)
np.sqrt(a)

# 矩陣乘法用 @ 或 np.matmul
A = np.ones((3,4)); B = np.ones((4,2))
A @ B           # shape (3,2)
# Broadcasting 規則：較小的 array 自動「廣播」對齊較大的
# Broadcasting rule: smaller array stretched to match larger

expr = np.ones((100, 5))         # 100 cells × 5 genes

# 每個 cell 加自己的 size factor / Add a per-cell size factor
size_factor = np.array([2, 3, 4] * 33 + [1])   # shape (100,)
normed = expr / size_factor[:, None]   # 變 (100,1) 才能廣播到 (100,5)

# 每個基因減自己的平均 (中心化) / Center each gene
gene_mean = expr.mean(axis=0)         # shape (5,)
centered = expr - gene_mean              # (100,5) - (5,) → 廣播

# 標準化 (z-score) 每個基因
gene_std = expr.std(axis=0)
zscore = (expr - gene_mean) / gene_std
x = rng.normal(0, 1, (100, 5))

# 純 NumPy 重現 scanpy 的 normalize_total + log1p
# Pure NumPy reimpl of scanpy's normalize_total + log1p

# 假資料：100 cells × 1000 genes，count 矩陣
counts = rng.poisson(2, (100, 1000)).astype(np.float32)

# Step 1: 每 cell 總 counts
total_per_cell = counts.sum(axis=1)            # (100,)

# Step 2: 縮放到統一深度 (e.g., 10000)
target = 10000.0
scale = target / total_per_cell             # (100,)
normed = counts * scale[:, None]              # broadcast

# Step 3: log(1 + x) — 變異穩定
log_normed = np.log1p(normed)               # 不會 log(0) 報錯

print(log_normed.shape, log_normed.mean())

📌
稀疏矩陣關鍵：scRNA-seq 一個 cell 表達 1000 / 30000 個基因，剩下 29000 都是 0。scipy.sparse.csr_matrix 只儲存非零值，記憶體從 GB 降到 MB。AnnData 預設也用稀疏矩陣存。 Sparse matrices matter: A scRNA-seq cell expresses 1000 / 30000 genes; the other 29000 are zero. scipy.sparse.csr_matrix stores only non-zero values — memory drops from GB to MB. AnnData uses sparse storage by default.
