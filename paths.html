檔案路徑與 I/O

FileNotFoundError 是 Python 新手最常見的錯誤 — 本章把路徑問題講到清楚。

FileNotFoundError is the #1 newbie error in Python — this chapter clears the path problem once and for all.

概念

一、為什麼路徑這麼容易出錯？

當你寫 open("data.csv")，Python 會去「當前工作目錄 (CWD, Current Working Directory)」找 data.csv。問題是：CWD 是什麼？這要看你怎麼啟動 Python：

  • 用終端機 cd /path/to/proj && python script.py → CWD = /path/to/proj
  • 在 VSCode 點「Run Python File」 → CWD 通常是專案根目錄（workspace 資料夾）
  • 在 Jupyter Notebook → CWD = notebook 所在的資料夾
  • 用 cron / systemd 排程 → CWD = / 或 /root

同一段 open("data.csv") 在不同情境會找到不同的 (或找不到) 檔案 — 這就是 90% 路徑錯誤的根源。

When you write open("data.csv"), Python looks for data.csv in the Current Working Directory (CWD). The catch: CWD depends on how you launched Python:

  • Terminal: cd /path/to/proj && python script.py → CWD = /path/to/proj
  • VSCode "Run Python File" → CWD usually = workspace root
  • Jupyter Notebook → CWD = the notebook's folder
  • cron / systemd jobs → CWD = / or /root

The same open("data.csv") finds (or fails to find) different files in different contexts — root cause of 90% of path errors.

🚨
遇到 FileNotFoundError 第一件事：印出 os.getcwd()，看 Python 現在以為自己在哪個資料夾。99% 你會發現 CWD 跟你想的不一樣。 First action on FileNotFoundError: print os.getcwd() to see where Python thinks it is. 99% of the time CWD differs from what you assumed.
路徑兩大類

二、絕對路徑 vs 相對路徑

 絕對路徑相對路徑
定義從根目錄寫起的完整位置相對於 CWD 的位置 DefinitionFull location from filesystem rootLocation relative to CWD
Windows 例C:\Users\charlene\proj\data.csvdata\sample.csv 或 ..\figs\out.png Windows ex.C:\Users\charlene\proj\data.csvdata\sample.csv or ..\figs\out.png
Linux/Mac 例/home/charlene/proj/data.csv./data/sample.csv Linux/Mac ex./home/charlene/proj/data.csv./data/sample.csv
優點100% 不會找錯位置專案搬家、給協作者用都還能跑 ProsAlways finds the filePortable across machines/collaborators

建議兩者結合：用 pathlib.Path(__file__).parent 找腳本位置，再用相對路徑接到 data/。最穩、最 portable。 RecommendedCombine both: use pathlib.Path(__file__).parent to anchor at the script's location, then relative paths down to data/. Most robust and portable.
⚠️
Windows 反斜線坑：Python 字串裡 \ 是「跳脫字元」，所以 "C:\new" 會被當成 C: + 換行 (\n) + ew！解法：
① 改用正斜線 "C:/Users/.../data.csv"（Windows 也吃這個），或
② 用 raw string r"C:\Users\..."（前面加 r 表示不解析跳脫字元），或
③ 用 pathlib.Path（推薦）。 Windows backslash trap: In Python strings, \ is the escape character, so "C:\new" becomes C: + newline (\n) + ew! Fixes:
① Use forward slashes "C:/Users/.../data.csv" (Windows accepts these), or
② raw string r"C:\Users\..." (the r disables escape parsing), or
③ pathlib.Path (recommended).
現代寫法

三、現代解法：pathlib (取代 os.path)

Python 3.4 起內建 pathlib，比舊版 os.path 更乾淨、更跨平台、操作更直覺。新專案請一律用 pathlib。

Python 3.4+ ships pathlib, which is cleaner, more cross-platform, and more intuitive than the old os.path. Always use pathlib for new projects.

四、open()、模式、編碼

讀寫檔案永遠用 with open(...) as f:，這個 with 區塊結束時會自動關閉檔案，避免忘記 close() 造成資料漏寫。

Always use with open(...) as f:. The with block auto-closes the file when it exits, so you can't forget close() and lose buffered writes.

決策引導

六、路徑寫法決策樹

🌳 如何選路徑寫法？

Q1:
是可重複執行的腳本嗎（要交給別人/丟到伺服器）？→ 是 → 用 Path(__file__).parent 為錨
Q2:
是互動式 Jupyter notebook？→ 是 → 用 Path.cwd()（notebook 的位置）；或在 cell 開頭 %cd /path/to/proj
Q3:
是大型 Snakemake / Nextflow pipeline？→ 是 → 用 pipeline 框架的內建路徑變數（config["data_dir"]）
Q4:
路徑要寫到 git？→ 一律相對 → 任何含使用者名稱的絕對路徑都不該進 git
Q1:
Is it a repeatable script (shared with others / on a server)? → Yes → Anchor with Path(__file__).parent
Q2:
Is it an interactive Jupyter notebook? → Yes → Use Path.cwd(), or %cd /path/to/proj in the first cell
Q3:
Is it a Snakemake / Nextflow pipeline? → Yes → Use the framework's path variables (config["data_dir"])
Q4:
Will the path be committed to git? → Always relative → Never commit paths containing user names
