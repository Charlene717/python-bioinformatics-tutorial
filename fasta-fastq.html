FASTA / FASTQ 處理

基因序列、定序 reads — 兩種你天天會碰的格式，從讀寫到 QC 一次學完。

Reference sequences and sequencing reads — the two formats you'll touch daily, from I/O to QC.

格式

一、FASTA vs FASTQ

 FASTA (.fa, .fasta)FASTQ (.fq, .fastq)
用途儲存「成品」序列：基因組、轉錄本、蛋白質儲存「原始」定序 reads (附品質分數) UseStores assembled sequences: genome, transcripts, proteinsStores raw sequencing reads (with quality scores)
每筆紀錄2 行：>ID + 序列4 行：@ID + 序列 + + + 品質字串 Per record2 lines: >ID + sequence4 lines: @ID + seq + + + quality
檔案大小小 (人類基因組 ~3 GB)大 (一個樣本可能 50–100 GB，常壓 .gz) File sizeSmall (human genome ~3 GB)Huge (one sample 50–100 GB, usually gzipped)

AAACATTTTCAGACCTATGGAAACTACTTCCTGAAAACAACGTTCTGTC
>ENST00000269307 TP53 transcript variant 2
ATGGAGGAGCCGCAGTCAGATCCTAGCGTCGAGTCCCAACTGAATCCTC
...
# 規則：
#  • 標頭行以 > 開始；> 後第一個空白前是 ID，之後是 description
#  • 後面的非 > 行都是序列，可以斷行也可以全部一行
#  • 一個 FASTA 檔可以含多筆紀錄
@SRR000001.1 length=36
TATTGGGAATTGCAACTTGGCAGTGTGTGGAATTTC
+
IIIIIIIIIIIIIIIIIIIIIIIIIIIIIB1@?B?-
@SRR000001.2 length=36
GTAGAGTGCAGAGTACGCATGCATCAATCAGGTACG
+
IIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIDDDDD
# 4 行為一筆：
#  Line 1: @ + read ID
#  Line 2: 序列 (DNA)
#  Line 3: + (可重複 ID 也可空)
#  Line 4: 品質分數 — 每個字元對應序列上一個 base 的品質
#
# 品質字元 → Phred score：ord(char) - 33  (Sanger / Illumina 1.8+)
#   'I' (73) - 33 = 40   → 99.99% 準確 (極好)
#   '!' (33) - 33 = 0    → 100% 機率錯 (最差)

# 過濾、改名後再寫出 / Filter and rewrite
def long_only(records, min_len=200):
    for r in records:
        if len(r.seq) >= min_len:
            r.description = ""
            yield r

records = SeqIO.parse("in.fa", "fasta")
SeqIO.write(long_only(records), "out.fa", "fasta")
# 用 generator 不會把所有東西讀進記憶體
from Bio import SeqIO

# FASTQ 比 FASTA 多了 letter_annotations["phred_quality"]
for rec in SeqIO.parse("reads.fastq", "fastq"):
    seq = rec.seq
    quals = rec.letter_annotations["phred_quality"]   # list of int
    avg = sum(quals) / len(quals)
    print(rec.id, avg)

# 過濾低品質 reads（前 50bp 平均 > 30）/ Filter low-Q reads
def good(records, min_avg=30, head=50):
    for r in records:
        q = r.letter_annotations["phred_quality"][:head]
        if q and sum(q) / len(q) >= min_avg:
            yield r

SeqIO.write(good(SeqIO.parse("in.fq", "fastq")),
            "clean.fq", "fastq")

# 截斷 / Trim reads to first 100 bp
def trim(records, length=100):
    for r in records:
        yield r[:length]   # 切片會帶著 quality 一起切！

QC 概念

四、QC 該看什麼？

📊 每位置品質

Illumina reads 的品質通常 5' 端 (read 開頭) 高、3' 端逐漸下降。如果中段就崩，表示 run 有問題。
工具：FastQC。

Illumina reads typically have high quality at 5', degrading toward 3'. If quality crashes mid-read, the run had issues.
Tool: FastQC.

🧬 adapter 污染

當 insert 太短，定序儀會讀進 adapter。要用 cutadapt / trim_galore / fastp 修剪。

Short inserts cause read-through into adapter. Trim with cutadapt / trim_galore / fastp.

📈 GC 分佈

正常物種 GC% 應該呈鐘型分佈。出現雙峰 → 可能有汙染（細菌、其他物種）。

Should be roughly bell-shaped. Bimodal → possible contamination (bacteria, other species).

🔁 重複序列

過高表示 PCR 過度放大。RNA-seq 通常 30–50% 重複是正常的；DNA-seq 應 <20%。

High = PCR over-amplification. RNA-seq normally 30–50%; DNA-seq should be <20%.
