BLAST 與序列比對

從一條未知序列找出它在資料庫的同源 — BLAST 的觀念、執行、結果解讀。

From an unknown sequence to its database homologs — BLAST concepts, execution, result interpretation.

概念

一、BLAST vs 其他比對工具

BLAST = Basic Local Alignment Search Tool。給一條序列，從上百萬條的資料庫中找出「相似」的序列，回傳一張表（誰相似、相似多少、機率值）。

BLAST 屬於區域 (local) 比對，找的是序列裡「相似的片段」。其他常見比對：

  • BLAST：找未知序列的同源（演化、註解）— 慢、敏感
  • BWA / Bowtie2：把短 reads 對到參考基因組（NGS pipeline）— 超快
  • Minimap2：長 reads（PacBio / Nanopore）— 快又能對長序列
  • MUSCLE / MAFFT：多序列比對（演化樹、結構功能比較）
  • pairwise2 (Biopython)：兩條序列比對，適合教學/小規模

BLAST = Basic Local Alignment Search Tool. Given a query sequence, find similar sequences in a multi-million record database; returns a table (who matched, how well, with what probability).

BLAST is local alignment — finds similar fragments. Other common aligners:

  • BLAST: homology search (annotation, evolution) — slow, sensitive
  • BWA / Bowtie2: short reads to reference (NGS pipelines) — very fast
  • Minimap2: long reads (PacBio / Nanopore)
  • MUSCLE / MAFFT: multiple sequence alignment (phylogenetics, structure-function)
  • pairwise2 (Biopython): two-sequence alignment, good for teaching/small scale

關鍵指標

二、E-value 與 bit score

📉 E-value

「在隨機資料庫中能純粹靠運氣得到這個分數的次數」。
越小越可信！
• E < 1e-10：非常可信
• E < 1e-5：通常可信
• E > 0.01：很可能是巧合

E-value 會隨著 DB 大小而變 — DB 越大、相同分數的 E-value 越大（噪音變多）。

"Number of hits this score would be expected by chance alone." Lower = better.
• E < 1e-10: highly significant
• E < 1e-5: usually meaningful
• E > 0.01: probably random

E-value depends on DB size — larger DB inflates E for the same score.

📊 bit score

標準化的相似度分數。不會受 DB 大小影響 — 跨資料庫比較選 bit score。
• > 200: 很強
• 50–80: 邊緣，謹慎

Normalized similarity score. Doesn't depend on DB size — use this when comparing across DBs.
• > 200: very strong
• 50–80: marginal, be careful

⚠️
陷阱：低 % identity 不代表沒同源！例如蛋白質可能 30% identity 但結構/功能完全保留（distant homolog）。同源學家通常用 bit score > 50 + 有意義的功能保守 motif 來下結論。 Pitfall: Low % identity does not mean no homology. Proteins at 30% identity can still share full structure/function (distant homologs). Experts typically combine bit score > 50 with conserved functional motifs to call homology.

三、用 Python 跑 BLAST

from Bio.Blast import NCBIWWW, NCBIXML
from Bio import SeqIO

# 線上 BLAST = 投到 NCBI 伺服器、慢、但不用裝資料庫
# Online = submit to NCBI server, slow, no local DB needed
# 適合：偶爾查 1–10 條序列；不適合大規模
# Good for: ad-hoc 1–10 queries; not for batch

record = SeqIO.read("my_query.fa", "fasta")
result_handle = NCBIWWW.qblast("blastn", "nt", record.seq,
                               hitlist_size=10, expect=1e-5)

# 把結果存下來（不要重跑！很慢）
with open("results/blast.xml", "w") as f:
    f.write(result_handle.read())

# 重新讀回 XML
with open("results/blast.xml") as f:
    blast_record = NCBIXML.read(f)
# 安裝 BLAST+ 命令列工具 / Install BLAST+
# conda install -c bioconda blast

# 1. 下載 / 自製資料庫
# makeblastdb -in my_proteins.fa -dbtype prot -out my_db

            print(f"   {hsp.sbjct[:60]}...")
# pairwise2: 兩條序列直接比對 (適合小規模/教學)
# For two sequences directly (small scale / teaching)
from Bio import pairwise2
from Bio.pairwise2 import format_alignment

a = "ACCGTAGTC"
b = "ACGTGTC"

# Global (Needleman-Wunsch)：對齊整段
alignments = pairwise2.align.globalxx(a, b)
print(format_alignment(*alignments[0]))

# Local (Smith-Waterman)：找最佳區域
alignments = pairwise2.align.localms(a, b, 2, -1, -2, -.5)
# 參數：match=2, mismatch=-1, gap_open=-2, gap_extend=-.5

# 蛋白質用 BLOSUM62 替換矩陣
from Bio.Align import substitution_matrices
matrix = substitution_matrices.load("BLOSUM62")
alignments = pairwise2.align.localds("PEEKSAVTAL", "PEKSALTAL",
                                     matrix, -10, -.5)

# 注意：pairwise2 在 Biopython 1.84+ 標記為 deprecated，新版改用 Bio.Align.PairwiseAligner
from Bio.Align import PairwiseAligner
aligner = PairwiseAligner()
aligner.mode = "local"; aligner.match_score=2; aligner.mismatch_score=-1
alignments = aligner.align(a, b)
print(next(iter(alignments)))
MSA

四、MSA & 演化樹

MSA 是同時對齊 3 條以上的序列。常用工具：MUSCLE、MAFFT、Clustal Omega。Python 通常是用 subprocess 呼叫，再用 Biopython 讀回。

MSA aligns 3+ sequences simultaneously. Common tools: MUSCLE, MAFFT, Clustal Omega. From Python, usually invoke via subprocess and read results back with Biopython.

import subprocess
from Bio import AlignIO, Phylo
from Bio.Phylo.TreeConstruction import DistanceCalculator, DistanceTreeConstructor

# 1. 跑 MAFFT (conda install -c bioconda mafft)
subprocess.run("mafft --auto data/raw/genes.fa > results/aligned.fa",
               shell=True, check=True)

# 2. 讀進來
aln = AlignIO.read("results/aligned.fa", "fasta")
print(aln)
print(f"{len(aln)} 條 × {aln.get_alignment_length()} bp")

# 3. 建距離樹 / Build NJ tree
calc = DistanceCalculator("identity")
dm = calc.get_distance(aln)
constructor = DistanceTreeConstructor()
tree = constructor.nj(dm)

# 4. 畫出來
import matplotlib.pyplot as plt
fig, ax = plt.subplots(figsize=(8,6))
Phylo.draw(tree, axes=ax, do_show=False)
fig.savefig("results/figures/tree.pdf")
決策引導

五、選對工具

🌳 比對工具決策樹

Q1:
有一條未知序列、想知道它「是什麼」？ → BLAST（線上 NCBI 即可）
Q2:
有上百萬條短 reads，要對到參考基因組？ → BWA-MEM2 或 STAR（RNA-seq）
Q3:
有 PacBio / Nanopore 長 reads？ → Minimap2
Q4:
想找跨物種同源，做演化分析？ → MAFFT 或 MUSCLE 做 MSA，再用 IQ-TREE / RAxML 建樹
Q5:
只想兩條序列對一下、看 identity？ → Bio.Align.PairwiseAligner
Q1:
One unknown sequence, want to know what it is? → BLAST (online NCBI is fine)
Q2:
Millions of short reads to align to a reference? → BWA-MEM2 or STAR (RNA-seq)
Q3:
PacBio / Nanopore long reads? → Minimap2
Q4:
Cross-species homology / phylogeny? → MAFFT or MUSCLE for MSA, then IQ-TREE / RAxML
Q5:
Just two sequences, identity check? → Bio.Align.PairwiseAligner
