環境設定與安裝

在動手寫一行 Python 之前，先把工作環境、資料夾、虛擬環境一次搞定 — 這是日後不踩坑的第一步。

Before writing a single line of Python, lock down your environment, folders, and virtual envs — the foundation that keeps you out of trouble later.

概覽

為什麼第一步是「環境」而不是程式？

很多生資新手第一次學 Python 卡關，不是因為程式邏輯太難，而是「跑不起來」：找不到檔案、套件版本衝突、Jupyter 開不起來、conda 跟 pip 打架……這些都是環境問題。

本章會花時間把這些細節說清楚：你會學到 Python 解譯器、套件管理器、虛擬環境、IDE 的角色分工，並且實際裝好一個能跑生資分析的乾淨環境。

核心概念：「Python 環境 = Python 版本 + 一堆套件 + 它們互相相容」。一台電腦上可以同時有很多個環境，每個專案用一個專屬環境是最佳實務。

Many bioinformatics newcomers get stuck on their first Python attempt — not because the logic is hard, but because nothing runs: file not found, package version clash, Jupyter won't start, conda fighting pip. All of these are environment issues.

This chapter takes the time to spell out the details. You'll learn the roles of the Python interpreter, package manager, virtual environment, and IDE, and you'll set up a clean environment ready for bioinformatics work.

Core concept: "A Python environment = a Python version + a set of packages + their mutual compatibility." A single machine can host many environments, and giving each project its own is best practice.

💡
一句話原則：「每個專案開一個 conda env，永遠別碰 system Python。」記住這句話，可以省下未來 90% 的痛苦。 Single-rule principle: "One conda env per project, never touch system Python." This single habit saves 90% of future pain.
核心元件

一、你需要安裝哪些東西？

🐍

Python 解譯器 (Interpreter)

把你寫的 .py 程式碼翻譯成電腦能執行的指令。生資分析建議使用 Python 3.10–3.12。
不要直接用 Python 官網的安裝包 — 改用 Miniconda 一次解決「Python + 套件管理」。

Translates your .py code into instructions the computer can run. For bioinformatics use Python 3.10–3.12.
Don't install Python from python.org directly — use Miniconda instead, which handles "Python + package manager" together.

📦

套件管理器 (conda / pip)

安裝、升級、移除套件 (numpy, pandas, scanpy…) 的工具。
• conda 也能管 Python 版本與非 Python 工具 (samtools, R)。
• pip 只管 Python 套件，但種類最齊全。
兩者可混用（先 conda 後 pip）。

Tool for installing, upgrading, removing packages (numpy, pandas, scanpy…).
• conda also manages Python versions and non-Python tools (samtools, R).
• pip handles Python-only packages but has the broadest catalog.
You can mix them (conda first, pip second).

🧪

虛擬環境 (env)

把不同專案的套件「隔離」開來，避免 A 專案的 numpy 1.20 和 B 專案的 numpy 1.26 互相打架。
常見方案：conda env（推薦）、venv、uv。

Isolates each project's packages so project A's numpy 1.20 doesn't clash with project B's numpy 1.26.
Common options: conda env (recommended), venv, uv.

💻

IDE / 編輯器

寫 Python 的環境。
• VSCode：最通用、外掛多、適合所有人。
• PyCharm：功能最強、適合大專案。
• JupyterLab：適合互動分析、看圖、生資論文。
新手建議：VSCode + Jupyter 外掛。

Where you write Python.
• VSCode — most versatile, huge extension ecosystem, fits everyone.
• PyCharm — most powerful, ideal for large projects.
• JupyterLab — best for interactive analysis, plots, papers.
Beginner pick: VSCode + Jupyter extension.

實作步驟

二、從零開始的完整安裝流程

下載並安裝 Miniconda

到 https://docs.conda.io/en/latest/miniconda.html 下載對應作業系統的 64-bit 版本。安裝時：

  • Windows：「Install for Just Me」、勾選「Add Miniconda3 to my PATH environment variable」（注意：會有警告，但對個人開發機建議勾）。
  • Mac/Linux：用 .sh 安裝檔，安裝時選 yes 來執行 conda init。

安裝完成後關閉所有終端機，重新開啟。

Download the 64-bit installer for your OS at https://docs.conda.io/en/latest/miniconda.html. During install:

  • Windows: "Install for Just Me", check "Add Miniconda3 to my PATH environment variable" (yes there's a warning, but for personal dev machines it's fine).
  • Mac/Linux: use the .sh script, answer yes to running conda init.

Close all terminals after install, then open a new one.

驗證安裝

打開終端機 (Windows: Win+R → cmd；Mac: Spotlight → Terminal)，輸入：

Open a terminal (Windows: Win+R → cmd; Mac: Spotlight → Terminal) and run:

$ conda --version conda 24.1.2 $ python --version Python 3.12.2

有看到版本號就 OK。如果出現「'conda' is not recognized」，表示 PATH 沒設好 — Windows 可改用「Anaconda Prompt」開始程式集裡的捷徑。

If you see version numbers, you're good. If you get "'conda' is not recognized", PATH isn't set — on Windows, use the "Anaconda Prompt" shortcut from the Start Menu instead.

建立第一個生資專用環境

幫這個課程建立一個叫 pybio、Python 3.11 的環境：

Create an env called pybio with Python 3.11 for this course:

$ conda create -n pybio python=3.11 -y $ conda activate pybio (pybio) $ # prompt 前出現 (pybio) 表示已啟用

安裝核心套件

分批安裝，避免 dependency resolver 卡住：

Install in stages so the dependency resolver doesn't choke:

# 1. 數值與資料 / numerics & data $ conda install -c conda-forge numpy pandas matplotlib seaborn scipy scikit-learn jupyterlab -y # 2. 生資基礎 / bioinformatics core $ conda install -c bioconda biopython pysam -y # 3. 單細胞 (用 pip 較新) / single-cell (pip is fresher) $ pip install scanpy anndata leidenalg python-igraph

安裝 VSCode 與 Python 外掛

到 https://code.visualstudio.com 下載安裝。打開後到 Extensions（左側方塊圖示）搜尋並安裝：

  • Python（Microsoft 官方）— 必裝
  • Pylance— 智慧型提示
  • Jupyter — 在 VSCode 內跑 notebook
  • Rainbow CSV— 看 CSV 顏色更清楚

Install VSCode from https://code.visualstudio.com. Open it and from Extensions (the squares icon) search and install:

  • Python (official Microsoft) — required
  • Pylance — smart IntelliSense
  • Jupyter — run notebooks inside VSCode
  • Rainbow CSV — colorize CSV files

在 VSCode 中選用 pybio 環境

開啟一個 .py 檔案後，按 Ctrl+Shift+P → 輸入 Python: Select Interpreter → 選 pybio。畫面右下角應顯示 3.11.x ('pybio')。

Open any .py file, press Ctrl+Shift+P → type Python: Select Interpreter → choose pybio. The bottom-right of the window should now show 3.11.x ('pybio').

資料夾規劃

三、生資專案的資料夾規劃 (極重要)

新手最常犯的錯：把所有 .py、.csv、.fastq、圖片全堆在桌面。三天後就找不到自己昨天分析了什麼。

建議結構（任何規模都適用）：

Most common newbie mistake: dumping all .py, .csv, .fastq and figures onto the Desktop. Three days later you can't tell what you analyzed yesterday.

Recommended layout (works at any scale):

my_project/ ├── data/ # 原始資料 — 絕對不修改！read-only │ ├── raw/ # 從定序儀/GEO 下載的原始檔 │ └── processed/ # 你跑出來的中間檔 (可重建) ├── results/ # 圖、表、最終輸出 │ ├── figures/ │ └── tables/ ├── notebooks/ # Jupyter notebooks (探索性分析) │ └── 01_qc.ipynb ├── scripts/ # 正式 .py 腳本 (可重複執行) │ └── run_qc.py ├── env.yml # conda 環境定義 (給別人重現你的環境) └── README.md # 專案說明
⚠️
三大鐵則：
① data/raw/ 一旦放進去就絕不修改、絕不刪除。所有清洗都產出到 processed/。
② 路徑用相對路徑（從專案根目錄開始），別寫 C:\Users\charlene\Desktop\...，否則換電腦就壞掉。
③ data/ 通常很大 — 用 .gitignore 排除，不要傳 git。 Three iron rules:
① Once a file lands in data/raw/, never modify or delete it. All cleaning produces output in processed/.
② Use relative paths (from project root). Never hardcode C:\Users\charlene\Desktop\... — it breaks on any other machine.
③ data/ is usually huge — exclude it with .gitignore, don't commit to git.
可重現性

四、保存環境與分享給協作者

生資論文一個重要要求：「別人能否在他的電腦上重現你的結果？」這就靠匯出環境定義檔。

A critical requirement in bioinformatics papers: "Can someone else reproduce your results on their machine?" This is what env files are for.

# 1. 在你的環境裡匯出 / Export your env spec
(pybio) $ conda env export --no-builds > env.yml

# env.yml 內容大概長這樣 / env.yml looks like:
name: pybio
channels:
  - conda-forge
  - bioconda
dependencies:
  - python=3.11
  - numpy=1.26.*
  - pandas=2.2.*
  - scanpy
  - pip:
    - leidenalg==0.10.2
# 2. 別人在他電腦重建環境 / Someone else recreates the env
$ conda env create -f env.yml
$ conda activate pybio
# 完全相同的套件版本！ / identical package versions!
# 純 pip 專案 / Pure pip projects
(pybio) $ pip freeze > requirements.txt
$ pip install -r requirements.txt
